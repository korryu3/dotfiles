# Code Reuse Review Agent

`gh pr diff`でPRの差分を取得し、変更内容について既存コードの再利用機会を検出する。

## 検出対象

- **既存ユーティリティとの重複**: 新しく書かれた関数がコードベース内の既存ユーティリティやヘルパーと同じ機能を持つ場合、既存の関数を使うよう指摘する
- **インラインロジックの置き換え**: 手書きの文字列操作、パス処理、環境変数チェック、型ガード等、既存ユーティリティで置き換え可能なインラインロジックを検出する

## 検索戦略

ユーティリティディレクトリ、共有モジュール、隣接ファイルなどを積極的にGrep・Globで検索し、再利用可能な既存コードを見つけること。推測ではなく、実際にコードベースを検索して裏付けを取ること。

## 出力形式

```
- `utils/string.ts:12` — sanitizeInput()が既に存在する。api/handler.ts:45のインライン実装は既存関数で置き換え可能
- `api/handler.ts:78` — パス結合を手書きしているが、lib/paths.tsにjoinSafe()がある
```
